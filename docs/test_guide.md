# CalcPad 测试指南

## 0. 测试前准备

### 硬件清单

| 物品 | 说明 |
|------|------|
| SuperMini nRF52840 | 开发板 |
| SSD1306 OLED | 128×64，4 针 I2C（GND VCC SCL SDA） |
| 杜邦线若干 | 用于短接测试 |
| USB 数据线 | 连接电脑（必须是数据线，不能是纯充电线） |

### OLED 接线

| OLED 针脚 | 接到开发板 | 引脚编号 |
|-----------|-----------|---------|
| GND | GND | - |
| VCC | VCC (3.3V) | - |
| SCL | D3 | P0.20 |
| SDA | D2 | P0.17 |

> **重要**：VCC 接 3.3V，不要接 5V（除非 OLED 模块自带稳压）。

### 矩阵引脚参考

```
行线 (Row)：
  Row 0 = P0.06 (D1)
  Row 1 = P0.08 (D0)
  Row 2 = P0.22 (D4)
  Row 3 = P0.24 (D5)
  Row 4 = P1.00 (D6)

列线 (Col)：
  Col 0 = P0.11 (D7)
  Col 1 = P1.04 (D8)
  Col 2 = P1.06 (D9)
  Col 3 = P0.31 (D19)
  Col 4 = P0.09 (D18)
```

## 1. 固件烧录验证

### 步骤

1. 双击开发板 RESET 按钮，进入 UF2 Bootloader（电脑会弹出一个 USB 磁盘）
2. 将编译好的 `.uf2` 固件拖入该磁盘
3. 开发板自动重启

### 预期结果

- [ ] 开发板 LED 正常闪烁（表示固件正在运行）
- [ ] USB 连接电脑后，系统识别为 HID 键盘设备

### 排查

如果电脑不识别键盘：
- 确认使用的是**数据线**（不是纯充电线）
- 尝试换一个 USB 口
- 在 Windows 设备管理器 → "键盘" 中查看是否有 "CalcPad"

---

## 2. OLED 显示屏测试

### 步骤

1. 按上方接线表连接 OLED（GND、VCC、SCL、SDA）
2. 给开发板上电（插 USB 或供电）
3. 等待约 **2-3 秒**（固件启动后延迟初始化显示）

### 预期结果

- [ ] OLED 亮起，显示如下界面：

```
[ CalcPad ]
--------------------

Ready
```

### 排查（如果屏幕不亮）

**检查项 1 — I2C 地址**：
- 大多数 SSD1306 模块默认地址是 `0x3C`
- 少数模块（背面有焊盘标注 ADDR）地址是 `0x3D`
- 如果是 `0x3D`，需要修改 `breadboard_test.overlay` 中的 `reg = <0x3c>` 为 `reg = <0x3d>`

**检查项 2 — 分辨率**：
- 确认 OLED 是 128×64 还是 128×32
- 如果是 128×32，需修改 overlay：
  - `height = <32>`
  - `multiplex-ratio = <31>`
  - 添加 `com-sequential;`

**检查项 3 — 接线**：
- SCL 和 SDA 不要接反
- GND 必须连接（共地）
- 确认焊点/杜邦线接触良好

**检查项 4 — 供电**：
- OLED 模块的 VCC 接 3.3V（开发板 VCC 引脚）

> **操作方法**：每次短接一组引脚（列线 + 行线），触碰即松开，不要长按。

### 预期行为

- 电脑记事本中出现对应的字符
- OLED 屏幕同步显示按下的键名

### 排查（如果按键无反应）

1. **确认 USB 连接**：
   - Windows 设备管理器 → 键盘 → 应看到新设备
   - 如果看不到，确认是数据线不是充电线

2. **确认引脚对应**：
   - 必须是**一根列线 + 一根行线**，不能两根都是行线或都是列线
   - 参考上面的引脚表确认 D 编号对应的引脚位置

3. **蓝牙测试**（如果 USB 不方便）：
   - 手机或电脑蓝牙设置中搜索 "CalcPad"
   - 配对连接后，再测试按键

## 4. 计算器模式测试

### 进入计算器模式

1. 短接 **P0.11 (D7) + P0.06 (D1)**（位置 0,0 = CALC 键）
2. OLED 应切换为：

```
[ CALC ]
--------------------

0
```

### 测试计算

| 操作 | 短接引脚 | OLED 预期变化 |
|------|----------|-------------|
| 输入 1 | D7 + D5 | 表达式显示 `1` |
| 输入 + | D19 + D4 | 表达式显示 `1+` |
| 输入 2 | D8 + D5 | 表达式显示 `1+2` |
| 按 = | D19 + D5 | 结果显示 `= 3` |
| 清空 | D19 + D6 | 回到 `0` |

### 完整算式测试

| 算式 | 按键序列 | 预期结果 |
|------|---------|---------|
| 7+3 | 7 → + → 3 → = | = 10 |
| 9*6 | 9 → * → 6 → = | = 54 |
| 8/2 | 8 → / → 2 → = | = 4 |
| 10-3 | 1 → 0 → - → 3 → = | = 7 |
| 10%3 | 1 → 0 → % → 3 → = | = 1 |
| 2+3*4 | 2 → + → 3 → * → 4 → = | = 14（乘法优先） |
| 1/0 | 1 → / → 0 → = | Error |

### 退出计算器模式

1. 再次短接 **P0.11 (D7) + P0.06 (D1)**
2. OLED 应恢复为：

```
[ CalcPad ]
--------------------

Ready
```

3. 此时按数字键应恢复为正常键盘输出

### 通过标准

- [ ] 按 CALC 键进入计算器模式，OLED 标题变为 `[ CALC ]`
- [ ] 数字输入正确显示在 OLED 表达式区
- [ ] 运算符正确追加到表达式
- [ ] 按 = 显示正确结果
- [ ] 2+3*4 = 14（运算符优先级正确）
- [ ] 1/0 显示 Error
- [ ] 按 BKSP 删除最后一个字符
- [ ] 按 CLEAR 清空表达式
- [ ] 再按 CALC 退出计算器，OLED 恢复普通模式
- [ ] 退出后按键正常输出到电脑

## 5. 常见问题

### Q: OLED 接上后一直黑屏

A: 按以下顺序排查：
1. 检查 VCC 是否接到 3.3V，GND 是否接好
2. 检查 SCL/SDA 是否接反
3. 确认模块 I2C 地址（0x3C 还是 0x3D）
4. 确认分辨率是 128×64 还是 128×32
5. 换一根数据线/换一个 OLED 模块测试

### Q: USB 连接后电脑不识别键盘

A: 检查：
1. 必须使用数据线（不是纯充电线）
2. 尝试其他 USB 口
3. 双击 RESET 重新烧录固件
4. 检查 Windows 设备管理器中是否有未知设备

### Q: 蓝牙搜不到 CalcPad

A: 检查：
1. 固件是否正确烧录（开发板 LED 是否在闪）
2. 手机/电脑蓝牙是否已开启
3. 如果之前配对过其他固件，需要在蓝牙设置中删除旧配对记录

### Q: 按键输出的字符不对

A: 检查：
1. 确认短接的是一根行线 + 一根列线（参考引脚表）
2. 注意 NumLock 状态：Keypad 数字键需要 NumLock 开启才输出数字
3. 测试时建议先打开系统 NumLock
